# Sアセス問題生成ルール

## 目的

2027年の一次試験対策として、テキスト通読の代替ではなく、問題演習から必要箇所を読み返す学習導線を作る。

## 追加先

- 問題データ: `src/data/questions.ts` (既存分) と `src/data/questions-<member|roumu|keisu>-add.ts` (追加分。questions.ts が結合してエクスポートする。並列作問時は科目別ファイルだけを触る)
- 教科書本文: `source-review/cleaned/*.md`
- 法令確認メモ: `source-review/law-audit.md`
- 計数確認メモ: `source-review/calculation-audit.md`
- 出題傾向・実試験の確定正誤ポイント: `source-review/report-reference.md` (JMAMレポート採点結果+公式解説由来。論点選定・誤り選択肢パターン・検証済み計算公式として教科書本文と併用する)

## 作問の基本ルール

- 校閲済みMarkdownを根拠にする。
- 問題には必ず `sourceRef.subjectId`、`sourceRef.heading`、`sourceRef.quote` を入れる。
- `sourceRef.heading` は教科書ジャンプに使うため、校閲済みMarkdown内の見出し表記と一致させる。
- 原則として4肢の正誤型にする。正しいもの、または誤っているものをすべて選ばせる形式にし、正答数は1つまたは2つを基本にする。
- 問題文では正答数を明かさない。「1つ選べ」「2つ選べ」「最も適切なもの」など、単一正答を示す表現は使わない。
- データは `correctIndexes` に正答の選択肢番号を配列で入れる。単一正答でも `correctIndexes: [1]` のように配列で持つ。
- `type` は原則 `trueFalse` を使う。ケース判断でも、回答操作は4肢正誤の複数選択に揃える。
- 1つの問題で問う論点は1つに絞る。
- 解説は、正答理由だけでなく、誤答選択肢がなぜ違うかを必要に応じて補足する。
- 同じ見出しから複数問を作る場合は、定義、判断基準、例外、数値、ケース判断に分ける。

## 科目別注意点

### メンバー指導

- 教材固有の語句、順序、分類を優先する。
- 一般論に寄せすぎない。
- ケース問題は、教材本文のCaseと解説の対応関係を崩さない。

### 労務管理

- 電子教科書本文としては校閲版を使ってよい。
- 正誤判定に直結する数値、義務、期限、対象者要件は、問題化前に `law-audit.md` を確認する。
- 短時間労働者の社会保険は2026年10月予定の変更があるため、2027年用の問題生成前に再確認する。

### 計数入門

- 式、単位、表の数値を優先して確認する。
- 計算問題は、問題文、途中式、正答、解説の単位を揃える。
- 原価、固定費、変動費、埋没原価は、意思決定の文脈を明示して出題する。

## 追加後の検証

```bash
npm run audit:questions
npm run build
```

`audit:questions` は、重複ID、短すぎる解説、`correctIndexes` の妥当性、`sourceRef.heading` が校閲済みMarkdownに存在するかを確認する。
