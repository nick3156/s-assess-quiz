## 優先監査対象

`npm run audit:textbook` で検出される「本文量が薄い節」を優先的にスキャン照合する。

特に優先度が高い領域:

- メンバー指導 第3章後半から第4章
  - ほめ方、叱り方、質問、信頼関係の章に、説明文や補足欄の欠落が疑われる。
- メンバー指導の箇条書きのみの節
  - 原本では箇条書きの前後に説明文、POINT、ADVICE、Case振り返りがある可能性が高い。
- 労務管理 終盤のケース・演習系の節
  - Markdownが設問要約だけになっている箇所は、原本の解説や判断理由が抜けている可能性がある。
- 計数入門
  - `計数①.pdf`〜`計数④.pdf` を追加済み。
  - 表、グラフ、計算式、単位、例題の数値は省略せず、Markdown表・数式・文章説明として復元する。

## 運用変更

- 問題生成に使える本文は、スキャン照合済みの節だけに限定する。
- 章全体を「校閲済み」とみなさない。節単位で照合状態を管理する。
- OCRは補助に留め、PDF画像を一次ソースにする。
- 照合時は本文だけでなく、POINT、Case、ADVICE、図表、欄外注も必要に応じて教材本文へ統合する。
- 薄い節は「要約」ではなく「欠落候補」として扱う。

## 監査コマンド

```bash
npm run audit:textbook
```

このコマンドは、Markdown本文量・実質行数・箇条書きのみの節を機械的に検出する。検出結果は確定欠落ではないが、スキャン照合の優先順位として扱う。
