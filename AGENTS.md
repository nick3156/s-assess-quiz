- 定期校閲は `source-review/proofreading-routine.md` に従う。OCR字面の単純照合ではなく、スキャンPDF、OCR、前後文脈、教材の用語体系を突合して、意味として正しい校閲版に整える。

## アプリ実装

- 現行アプリは React/Vite 実装。旧単一HTMLは `legacy-index.html` に保存している。
- スマホ中心のPWAとして設計する。主要導線は `今日 / 問題 / 教科書 / 進捗` の下タブ。
- 出題ロジック (日替わりセット・弱点抽出・選択肢シャッフル) は `src/lib/quiz.ts` に置く。日付シードで決定的にし、同じ日の再訪で出題・並びが変わらないようにする。
- 教科書リーダーの必須挙動: 上部ナビのスクロール方向自動格納、右端スクラブ目次 (ナイアガラ式)、章見出しは本文内の章区切りとして描画 (ヘッダーと重複させない)、検索は結果リストからセクションへジャンプ。
- 配色は `src/app.css` 冒頭のデザイントークン (`:root` 変数、ライト/ダーク両対応) だけを使う。ハードコードした色を増やさない。
- 教科書本文の `**強調**` はリーダーで蛍光マーカー風に描画される (段落・リスト・正式なMarkdown表セルが対象)。「|」や「→」を含む擬似図ブロックとコードフェンスは `<pre>` 描画になり太字は剥がされるため、そこに `**` を書かない。強調は重要用語の初出・定義箇所と正誤判定に効く数値に限定し、1セクション2〜4箇所を目安にする (塗りすぎない)。
- 教科書本文を内蔵するため、公開時は限定公開またはアクセス制限を前提にする。無制限の公開URLに本文を置かない。
- 問題にはカテゴリ、章、出典、重要度、解説、確認状態を持たせる。
- 問題は `src/data/questions.ts`、教科書は `src/data/textbooks.ts` と `source-review/cleaned/` を参照する。
- 問題の `sourceRef.heading` は、教科書ジャンプに使うため、校閲済み本文の見出しと対応させる。
- 出題形式は原則4肢の正誤複数選択にする。正しいもの、または誤っているものをすべて選ばせ、正答数は1つまたは2つを基本にし、解答表示まで正答数を明かさない。
- 問題データは `correctIndexes` 配列で正答を持つ。単一正答でも `correctIndexes: [n]` とし、旧式の `correctIndex` は使わない。
- 目標はテキスト通読の代替ではなく、解説を通じて必要箇所を読ませるアウトプット中心学習。
- 計数入門はスキャン原本から教科書化を進める。問題化は照合済みチャンクから始める。
- 日次問題生成は、校閲済みチャンクのみを根拠にし、型チェック・重複チェック・ビルド検証が通った場合だけ反映する。
- 問題追加・作り直しは `source-review/question-generation.md` に従う。
- 問題を追加したら `npm run audit:questions` と `npm run build` を必ず実行する。
- 労務管理の問題化では、本文全体を止める必要はないが、正誤判定に直結する数値・義務・期限・対象者要件だけ `source-review/law-audit.md` を確認する。
